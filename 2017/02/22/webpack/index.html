Webpack 是当下最热门的前端资源模块化管理和打包工具

Webpack概述

Webpack可以将许多松散的模块按照依赖和规则打包成符合生产环境部署的前端资源。通过loader的转化，任何形式的资源都可以视作模块，CommonJs模块、CSS、图片、LESS、ES6模块等。

Gulp的工作方式是在一个配置文件gulpfile.js中，通过指明对特定文件执行一定的任务，实现编译，压缩，组合等操作，Gulp会自动帮你完成这些操作。

Webpack的工作方式是从一个主入口文件中如main.js，开始分析项目中的所有依赖文件，使用相应的loader处理它们，最后打包为一个或多个浏览器可识别的js文件。

Webpack使用

项目初始化

使用node npm初始化一个项目，方便模块管理

npm init

此时你需要填写项目名称、描述、版本等，可以默认直接回车。然后会在项目根目录下形成package.json文件

安装Webpack

//全局安装webpack
npm i webpack -g
//项目中安装webpack
npm i webpack --save-dev

项目结构搭建

  1. 我们可以在项目下建立两个目录，source和dist，source用于存放开发的源代码，dist存放打包后上线部署的代码
  2. 在source目录下创建index.html，main.js 。index.js为生成展示页面的模板文件，main.js为入口文件，最后打包后会在dist目录下生成打包后的文件bundle.js和根据模板生成的页面文件index.js,并且在index.js中自动引入bundle.js

Webpack的配置

  1. 我们在项目根目录下新建webpack.config.js ，如果想要区分开发模式可发表模式，也可以新建Webpack.develop.config.js和Webpack.public.config.js 。

  2. webpack.config.js最后打包时可以在cli中直接使用webpack命令，其他命名的文件需要在package.json的脚本中配置相应命令，然后可以通过如下：

    //package.json
"scripts": {
    "dev": "webpack --config webpack.develop.config.js",
    "pub": "webpack --config webpack.publish.config.js -p",
    "server": "webpack-dev-server  --config webpack.develop.config.js --content-base src"
}

    然后可以通过npm dev进行打包操作

  3. webpack.config.js配置

    webpack配置主要分下面几个方面入口和出口 ，loader ，plugin ，devtool

    //引入相应的模块
const webpack = require('webpack')
const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
//作为模块导出
module.exports = {
  //1.webpack 入口和出口
  entry: path.resolve(__dirname, './src/main.js'),
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'bundle.js'
  },

  //2.webpack loader
  module: {
    rules: [
      //babel-loader，转化ES6代码为ES5
      {
        test: /\.js$/,
        exclude: /(node_modules|bower_components)/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['env']
          }
        }
      },
      //vue-loader，编译vue文件
      {
        test: /\.vue$/,
        use: {
          loader: 'vue-loader'
        }
      },
      //css-loader，编译css
      {
        test: /\.css$/,
        use: [
          'style-loader', 'css-loader'
        ]
      },
      // 使用url-loader加载器  图片
      {
        test: /\.(png|jpg|jpeg|gif)$/,
        use: 'url-loader?limit=8192&name=images/[name].[ext]'
      },
      // 使用url-loader加载器  字体
      {
        test: /\.(woff|svg|eot|ttf)\??.*$/,
        use: 'url-loader?limit=8192&name=fonts/[name].[ext]'
      }
    ]
  },

  //3.plugins
  plugins: [
    //分离第三方的插件，将第三方代码抽出
    new webpack.optimize.CommonsChunkPlugin({ name: 'vendors', filename: 'vendors.js' }),
    //自动生成html的插件，指定模板和需要引入的js
    new HtmlWebpackPlugin({
      template: './src/index.html',
      htmlWebpackPlugin: {
        "files": {
          "js": ["bundle.js", "vendors.js"]
        }
      },
      // html压缩
      minify: {
        removeComments: true,
        collapseWhitespace: true,
        removeAttributeQuotes: true
      }
    }),
    //热更新插件，和热更新服务一起使用，每次改动可以实时预览效果
    new webpack.HotModuleReplacementPlugin()
  ],
  //4.开发工具 热更新服务
  devtool: 'eval',
  devServer: {
    contentBase: path.join(__dirname, './src'), // 当前服务器监听的路径
    hot: true,  // 热更新
    port: 8000,  // 定义端口号
    host: 'localhost',
    open: true,    // 是否自动打开浏览器，
    openPage: ''
  }
}

常用功能介绍

loaders

loaders使webpack有调用外部脚本的能力，loaders一般用来语法的转换和编译，如label-loader将ES6语法转换为ES5，vue-loader将vue文件编译为js等

loaders需要单独通过npm安装，并且需要在webpack.config.js中的modules下配置，有如下几个方面：

  • test ：匹配处理文件的扩展名，形式为正则表达式
  • loader ：loader的名称
  • include/exclude ：手动添加需要处理的文件或忽略不需要处理的文件

  • query ：loader额外的设置选项

    ​

常用loaders

  1. babel-loader

    用来转化ES6以及jsx等语法

    module: {
  rules: [
    //babel-loader，转化ES6代码为ES5
    {
      test: /\.js$/,
      exclude: /(node_modules|bower_components)/,
      use: {
        loader: 'babel-loader',
        options: {
          presets: ['env']
        }
      }
    },
  ]
}

    ​

  2. css-loader和style-loader

    css-loader实现@import和url()功能，style-loader将计算后的样式添加到页面中,

    如果要实现css模块化，避免类名冲突，需要在css-loader中增加options

    module: {
  rules: [
    {
      test: /\.css$/,
      use: [
        {
            loader: "style-loader"
        }, 
        {
            loader: "css-loader",
            options:{modules:true}//css模块化
        }
      ]
    }
  ]
}

    ​

  3. postcss-loader和autoprefixer插件

    根据can i use规则自动添加前缀，webpack配置文件中添加postcss-loader，在根目录新建postcss.config.js

    //webpack.config.js
module: {
  rules: [
    {
      test: /\.css$/,
      use: [
        {
            loader: "style-loader"
        }, 
        {
            loader: "css-loader"
        }
      ]
    }
  ]
}

    //postcss.config.js
module.exports = {
    plugins: [
        require('autoprefixer')
    ]
}

    ​

  4. 当然如果你使用了CSS预编译，你可以添加less-loader ，sass-loader ，stylus-loader等

Plugins

插件用来扩展webpack功能，插件的使用同样需要npm安装，然后在wenpack.config.js中的plugins中配置

常用插件

  1. webpack.optimize.CommonsChunkPlugin

    分离第三方代码

    plugins:[
  new webpack.optimize.CommonsChunkPlugin({ name: 'vendors', filename: 'vendors.js' }),
]

    ​

  2. HtmlWebpackPlugin

    根据模板自动生成html,并引入相应js

    npm i html-webpack-plugin --save-dev

    const HtmlWebpackPlugin = require('html-webpack-plugin')
plugins:[
  //自动生成html的插件，指定模板和需要引入的js
  new HtmlWebpackPlugin({
    template: './src/index.html',
    htmlWebpackPlugin: {
      "files": {
        "js": ["bundle.js", "vendors.js"]
      }
    },
    // html压缩
    minify: {
      removeComments: true,
      collapseWhitespace: true,
      removeAttributeQuotes: true
    }
  }),
]

  3. HotModuleReplacementPlugin

    文件修改后进行热更新,需配合webpack-dev-server使用

    plugins:[
  new webpack.HotModuleReplacementPlugin()
],

Devtool调试

打包后的代码不容易跟踪调试，source maps可以帮我们解决，在webpack.config.js中配置

devtool: 'eval-source-map'

devtool有多种配置选项，如下：

devtool 描述
source-map 在一个单独的文件中产生一个完整且功能完全的文件。这个文件具有最好的source map，但是它会减慢打包速度
cheap-module-source-map 在一个单独的文件中生成一个不带列映射的map，不带列映射提高了打包速度，但是也使得浏览器开发者工具只能对应到具体的行，不能对应到具体的列（符号），会对调试造成不便
eval-source-map 使用eval打包源文件模块，在同一个文件中生成干净的完整的source map。这个选项可以在不影响构建速度的前提下生成完整的sourcemap，但是对打包后输出的JS文件的执行具有性能和安全的隐患。在开发阶段这是一个非常好的选项，在生产阶段则一定不要启用这个选项
cheap-module-eval-source-map 这是在打包文件时最快的生成source map的方法，生成的Source Map 会和打包后的JavaScript文件同行显示，没有列映射，和eval-source-map选项具有相似的缺点

devServer

开启服务，实现实时刷新等功能

npm i webpack-dev-server --save-dev

devServer: {
    contentBase: path.join(__dirname, './src'), // 当前服务器监听的路径
    hot: true,  // 热更新，也可以是inline:true
    port: 8000,  // 定义端口号
    host: 'localhost',
    open: true,    // 是否自动打开浏览器，
    openPage: ''
}
