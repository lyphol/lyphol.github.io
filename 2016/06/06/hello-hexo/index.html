利用hexo+github pages搭建一个属于自己的博客

快速开始

环境的安装

  1. 安装node
  2. 注册github账号，并创建github pages项目
  3. 安装git客户端

具体的步骤不在这里详述

hexo安装

全局安装hexo,如果速度慢的话可以用淘宝镜像

npm i hexo -g

创建blog项目文件夹，并初始化项目

hexo init

hexo配置

github远程仓库的关联，在项目根目录下找到_config.yml,修改下面配置项

deploy:
    type: git
    repo: https://github.com/username/username.github.io.git
    branch: master

静态资源生成

此时最简单的配置已经完成，可以生成博客看看效果

清空源目录，一般在更新设置更换主题后重新生成时需要

hexo clean

页面生成，更新文章后执行此命令

hexo generate

本地预览页面，可以安装本地服务hexo-server

npm i hexo-server

推送远程仓库

本地预览没问题的话就可以推送远程仓库了，此时你需要先安装部署插件，本人在安装此插件时用淘宝镜像会报错，切换回npm镜像后解决

npm i hexo-deployer-git

然后就可以部署到github了

hexo depoly

最后就可以打开name.github.io查看你的博客了

新建文章

hexo new 'aticle name'

你可以使用上面的命令来新建一篇文章，名字最好为英文，本人测试时中文没有正确创建列表

所有的文章都在项目文件夹下的source/_posts目录下，你可以根据下表来自定义文章的标题、标签、创作时间等信息

参数 描述 默认值
layout 布局 post
title 标题 文件名
date 建立日期 文件建立日期
updated 更新日期 文件更新日期
tags 标签（不适用于分页）
categories 分类（不适用于分页）
permalink 覆盖文章网址
thumbnail 缩略图地址
toc 显示 TOC 按钮 true
comment 显示评论 true
notag 不生成标签按钮 false
top 置顶 false
mathJax 启用 Mathjax false

缩略图功能

在 Material 主题中，每个 Scheme 都有缩略图功能。
只需要在 Front-matter 中添加参数 thumbnail:，然后填入缩略图地址即可。

Paradox

此 Scheme 如果没有自定义缩略图，则使用默认随机缩略图，随机缩略图目录位于主题文件夹下 source -> img -> random。
随机缩略图可添加自己喜好的图片，格式为 <num>.png 。然后在 主题配置文件 中 thumbnail:random_amount 修改缩略图数量。

Isolation

此 Scheme 只会显示已自定义缩略图。

进阶设置

主要以hexo-theme-material主题为例，不同的主题具体设置可能不同

主题设置

如果你觉得自带的主题不好看的话，你可以更换自己喜欢的主题，博主用的主题为

hexo-theme-material，该主题符合Google Material Design规范，在PC端和移动端都有很好的效果，推荐一波

将主题文件下载放到项目文件夹下的theme下，然后修改hexo配置文件_config.yml文件中的theme: material项就可以可，当然主题也有一些可以自定义的选项，在后面会说到

设置完成后执行下面命令就可以了

hexo clean
hexo generate
hexo deploy

独立页面的添加

用于设置独立页面的入口，默认为空，请按照如下样例添加个人独立页面。

pages:
    About:
        link: "/about"
        icon: person
        divider: false
  • About 是该独立页面的名称，请自行修改。
  • link 的参数为相对路径，对应 hexo 目录下的 source 文件夹内的相应文件夹。
  • icon 的参数为自定义的 Material 图标，可用图标可在 Material icons 查询。
  • divider 设置成 true 后会在该条目底部增加一条分割线

关于更多页面的添加可以关注主题文档material
